A Non-Intrusive Approach To Measuring Flow Rate In A Pipe Using Machine Learning

Overview:

This project presents a smart, non-intrusive solution for measuring water flow rate in domestic pipelines using accelerometer data and machine learning. Instead of relying on intrusive sensors (Hall-effect, turbine, or ultrasonic meters), vibrations generated by flowing water are captured via accelerometers attached externally to the pipe. These vibration signals are then processed and mapped to flow rates using machine learning models (Regression, Decision Tree, Random Forest, DNN, and CNN).



Problem Statement

Conventional flow meters face major challenges:

  • Intrusiveness – require cutting into the pipe, increasing leakage risk.
  • Cost – high initial purchase and maintenance.
  • Scalability – difficult to deploy widely in low-resource settings.

Research Question: Can we accurately estimate water flow rates from vibration signals alone, using machine learning, in a low-cost, non-intrusive way?


Personal Contributions

  • Designed and implemented the non-intrusive sensing setup (Arduino Nano BLE + accelerometers + flowmeter for ground truth).
  • Developed the data acquisition pipeline and collected over 60k+ vibration-flow records.
  • Preprocessed and engineered features from accelerometer signals (time & frequency domain).
  • Trained and evaluated multiple ML/DL models (Regression, Decision Tree, Random Forest, DNN, 1-D CNN).
  • Demonstrated CNN as the most effective model (R² = 0.93) for flow rate prediction.
  • Drafted and co-authored the research paper (IEEE submission in progress).


System architecture (three subsystems)

1. Data Acquisition Subsystem

  • Sensors used:
    • Arduino Nano BLE Sense (with LSM9DS1 IMU)
    • External ADXL335 accelerometer
    • YF-S201 Hall-effect flowmeter (used only as ground-truth reference during training).
  • Setup:
    • Sensors attached externally to pipes of different lengths (0.5m, 1.0m, 2.5m).
    • Vibration data recorded at multiple flow rates between 6.57 – 25.80 L/min.
  • Volume of data:
    • Up to 69,000 records per experimental run, across different flow rates and pipe lengths.

2. Preprocessing & Feature Extraction

  • Vibration signals (accelerometer X, Y, Z axes) combined with reference flow rates.
  • Noise filtered, data normalized.
  • Prepared datasets split into train, test, and validation sets.

3. Machine Learning Subsystem

  • Models explored:
    • Traditional ML: Linear Regression, Decision Tree, Random Forest.
    • Deep Learning: DNN, 1-D CNN.
  • Tools: Google Colab + TensorFlow.
  • Performance metrics: R², MAE, MSE, RMSE.

Results

  • Traditional ML
    • Linear Regression: Poor fit, R² ≈ 0.01 → could not model non-linear patterns.
    • Decision Tree & Random Forest:
      • When accelerometer datasets combined → R² up to 0.998.
      • Captured strong correlations between vibration patterns and flow rates.
    • Deep Learning Models
      • DNN (3-layer fully connected network):
        • R² = 0.762
        • MAE = 0.922
        • Learned vibration-to-flow patterns but struggled with noise.
      • 1-D CNN (Convolutional Neural Network):
        • Best performing model.
        • R² = 0.9309
        • MAE = 0.4001, RMSE = 0.6325.
        • CNN captured temporal patterns in accelerometer signals effectively.
